### 참조 구현

GearUp 쪽 복제 런타임(gearup 저장소 `apps/demo/runtime/rhwp-bootstrap-runtime.ts`)이
동일 구조를 `bootstrap(rootEl, opts)` 형태로 이미 재조립해 사용 중 — opts/핸들
표면의 실전 검증본이므로 역이식으로 diff 를 최소화한다.

- opts (검증됨): `enablePostMessageApi`, `extendContextMenuItems({x,y,items,context})`
- 반환 핸들 (검증됨): wasm·extensionAPI·command factory 4종(InsertText/
  SplitParagraph/InsertTable/InsertPicture)·ready·getInputHandler·getCanvasView·
  loadFile·loadBytes·createNewDocument·destroy
- embed 정책 (신규 필요): 최소 줌 하한(grid 진입 차단)·모바일 자동 폭맞춤 off

## 구현 계획

### 1단계: bootstrap 추출 (동작 불변)
- `src/main.ts` 초기화 로직 전체를 `src/bootstrap.ts` 의
  `export function bootstrap(rootEl: HTMLElement, opts?: BootstrapOpts)` 로 이동.
- DOM 접근은 rootEl 스코프 query 로 통일 (현재 document.getElementById —
  단일 인스턴스 전제는 유지하되 스코프만 명확화).
- `main.ts` 는 `bootstrap(document.getElementById('app')!)` 호출로 축소 —
  스탠드얼론 앱 동작·번들 불변.
- 검증: 기존 E2E(text-flow) + 수동 스모크 (파일 열기·표 편집·줌).

### 2단계: embed opts + 반환 핸들
- `BootstrapOpts`: `enablePostMessageApi?`(기본 true)·
  `extendContextMenuItems?`·`embed?: { minZoom?: number; disableMobileAutoFit?: boolean }`.
- minZoom: ViewportManager.setZoom 하한 적용. disableMobileAutoFit:
  canvas-view 모바일 폭맞춤 분기 스킵 플래그.
- 반환 핸들: 참조 구현 표면 그대로 (위 인벤토리).
- 검증: 스탠드얼론(opts 미지정) 회귀 없음 + 임베드 스모크(호스트 저장소에서
  vite alias 전환 후 표 셀 클릭/선택/삽입).

### 3단계: 표/그림 spec 삽입 command factory 이관
- GearUp 쪽 스냅샷 기반 InsertTableCommand/InsertPictureCommand(+ insertTableSpec/
  insertPictureSpec, 약 300줄)를 `src/engine/embed-commands.ts` 로 upstream 이관.
- bootstrap 반환 핸들의 createInsertTableCommand/createInsertPictureCommand 가
  이를 사용.
- 검증: 표/그림 삽입 후 undo/redo·저장 라운드트립.

### 제약
- 스탠드얼론 앱(index.html 직접 서빙) 동작·UI 완전 불변.
- 기존 postMessage API 계약(hwpctl-load/rhwp-response) 불변.
- 렌더링/편집 엔진 로직 무변경 — 초기화 배선의 재배치만.
